GTA 6 Price & Countdown Tracker 网站设计文档

本文档用于指导 Agent 开发一个围绕 GTA 6 的英文 SEO 小站。网站不是新闻聚合站，而是一个 价格追踪 + 发售倒计时 + 预购状态 + 历史数据可视化 + 用户投票 的轻量级数据/工具站。

当前公开事实：Rockstar 官方公布 GTA VI 发售日期为 2026-11-19。价格、正式预购、各版本信息在未获得官方或平台商店确认前，统一显示为 “Not officially announced”。

1. 项目概述

项目名称

GTA 6 Price & Countdown Tracker

备选名称：

  • GTA 6 Tracker
  • GTA VI Price Tracker
  • GTA 6 Countdown Hub
  • Next Game Tracker：后续可扩展成大型游戏发售追踪站

一句话说明

一个非官方的 GTA 6 发售与价格追踪工具站，帮助玩家快速查看 GTA 6 发售倒计时、价格是否公布、预购是否开启、各平台状态、历史 GTA 系列数据和社区预测。

目标用户

  • 正在搜索 GTA 6 价格、发售时间、预购状态的普通玩家。
  • 想知道 GTA 6 是否开放预购的主机玩家。
  • 关注 GTA 6 价格是否会涨到 80/100 美元的游戏玩家。
  • 想查看 GTA 系列历史发售间隔、销量、评分、Trailer 数据的内容消费者。
重要合规提示： 本站必须明确标注 “Unofficial fan-made tracker. Not affiliated with Rockstar Games or Take-Two Interactive.”。 不要使用 Rockstar 官方 Logo、受版权保护的游戏截图、盗版资源、未授权品牌视觉素材。

2. 产品定位与差异化

2.1 不做什么

  • 不做纯新闻聚合站。
  • 不搬运游戏媒体文章。
  • 不发布未经确认的价格传闻作为事实。
  • 不做盗版下载、泄露内容、外挂、账号交易、CDKey 灰色导流。

2.2 要做什么

站点应该以结构化信息和工具价值为核心。核心卖点不是“我也报道 GTA 6 新闻”，而是：

  • 用户打开页面立刻看到倒计时。
  • 用户立刻知道官方价格是否公布。
  • 用户立刻知道 PS5 / Xbox / PC 各平台状态。
  • 用户可以查看不同地区、不同平台的预购状态。
  • 用户可以看到 GTA 系列历史发售间隔和数据可视化。
  • 用户可以投票预测价格，增加互动和停留时间。

2.3 核心差异化

普通游戏新闻站 本站差异化
写文章报道价格传闻 做价格状态追踪表，明确区分官方确认、平台占位、传闻
写“GTA 6什么时候发售”文章 做实时倒计时 + 时区说明 + 平台发售状态
写“GTA 6值得期待吗” 做 GTA 系列历史发售间隔、评分、销量、Trailer 数据图表
追逐新闻流量 沉淀 evergreen 数据页和工具页

3. MVP 范围

第一版目标是快速上线，抢占搜索引擎收录窗口。不要做复杂后端，不要做完整社区，不要做用户系统。

3.1 第一版必须包含

  • 英文站点。
  • 响应式设计，优先适配移动端。
  • 首页：倒计时 + 价格状态 + 预购状态 + FAQ。
  • 价格页：不同版本、平台、地区价格状态。
  • 预购页：不同平台预购状态。
  • 倒计时页：发售倒计时。
  • 历史数据页：GTA 系列发售时间线和数据可视化。
  • 投票模块：价格预测 / 平台选择，可先用本地模拟或第三方表单。
  • SEO 标题、描述、canonical、Open Graph、结构化数据。

3.2 第一版不做

  • 用户注册登录。
  • 评论系统。
  • 复杂后台。
  • 自动爬取所有电商平台价格。
  • 多语言。
  • 推送通知 App。

3.3 后续可扩展

  • 邮件提醒：价格公布 / 预购开启时通知。
  • 更多游戏：The Elder Scrolls VI、Witcher 4、Marvel’s Wolverine 等。
  • 自动化监控 PlayStation Store、Xbox Store、Amazon、Best Buy 等页面状态。
  • 社区投票结果长期保存。
  • Affiliate 商品推荐模块。

4. 页面与路由设计

路由 页面名称 主要关键词 页面目标
/ 首页 GTA 6 countdown, GTA 6 price, GTA 6 preorder 聚合展示最重要状态，承接主流搜索流量
/gta-6-price 价格追踪页 GTA 6 price, GTA VI price, how much is GTA 6 回答价格是否公布、各版本价格、各平台价格
/gta-6-preorder 预购追踪页 GTA 6 preorder, GTA 6 pre order PS5, GTA 6 preorder Xbox 回答预购是否开启、在哪里预购
/gta-6-countdown 倒计时页 GTA 6 countdown, how many days until GTA 6 提供实时发售倒计时
/gta-6-history 历史数据页 GTA release timeline, GTA games history, GTA 5 to GTA 6 gap 用可视化展示系列历史数据，提高差异化
/about 关于本站 unofficial GTA 6 tracker 说明非官方身份、数据来源、免责声明

4.1 首页结构

  1. Hero 区：GTA 6 Countdown + Release Date + Price Status。
  2. 核心状态卡片：Release Date、Official Price、Pre-order、Platforms、PC Version。
  3. 倒计时组件。
  4. 价格状态表。
  5. 预购状态表。
  6. 价格预测投票。
  7. Latest verified updates 时间线。
  8. FAQ。
  9. 免责声明。

4.2 价格页结构

  1. 标题：GTA 6 Price Tracker: Is the Official Price Announced?
  2. 结论卡：Official price has not been announced yet.
  3. 版本价格表：Standard / Deluxe / Premium / Collector’s Edition。
  4. 平台价格表：PS5 / Xbox Series X|S / PC。
  5. 地区价格表：US / UK / EU / Canada / Australia / India。
  6. 传闻处理模块：Rumored prices vs official information。
  7. 历史 AAA 游戏定价对比。
  8. FAQ。

4.3 预购页结构

  1. 标题：GTA 6 Pre-order Tracker: Is Pre-order Live?
  2. 结论卡：Pre-orders are not officially live yet.
  3. 平台表：PlayStation Store、Xbox Store、Amazon、Best Buy、GameStop 等。
  4. 按钮状态：Not live / Wishlist / Placeholder / Live。
  5. 邮件提醒入口：Get notified when pre-orders go live。
  6. FAQ。

4.4 倒计时页结构

  1. 大号倒计时：days / hours / minutes / seconds。
  2. 发售日期说明：November 19, 2026。
  3. 不同时区的发售日期说明。
  4. 平台发售状态。
  5. FAQ：Will GTA 6 come to PC? Is GTA 6 delayed?

4.5 历史数据页结构

  1. GTA 系列主要作品发售时间线。
  2. GTA V 到 GTA VI 等待时间可视化。
  3. GTA 系列平台变化。
  4. GTA 系列评分表。
  5. GTA 系列销量/出货量数据，若没有稳定来源，先只做可维护字段。
  6. Trailer 数据对比，注意数据需标注来源和更新时间。

5. 核心功能模块

5.1 倒计时模块

目标日期：2026-11-19T00:00:00，默认按用户本地时区显示。

  • 显示 Days / Hours / Minutes / Seconds。
  • 倒计时实时刷新。
  • 发售后自动切换为 “GTA 6 is out now”。

5.2 价格状态模块

  • 官方未公布前，显示 “Not officially announced”。
  • 不要把 Reddit、论坛、电商占位价写成官方价格。
  • 价格字段支持 confirmed / placeholder / rumor 三种状态。

5.3 预购状态模块

  • 展示各平台状态。
  • 状态包括 Not live、Wishlist、Placeholder、Live、Sold out。
  • 每条记录显示更新时间。

5.4 历史数据可视化

  • 使用 Chart.js、ECharts 或 Recharts。
  • 第一版建议只做 2 张图：发售时间线、作品间隔。
  • 图表下方保留数据表，利于 SEO。

5.5 投票模块

  • 问题 1：What do you think GTA 6 will cost?
  • 选项：$69.99 / $79.99 / $89.99 / $99.99+。
  • 问题 2：Which platform will you play on?
  • 第一版可用本地假数据，后续接 Supabase / Firebase。

5.6 邮件提醒模块

  • 第一版可只做表单 UI。
  • 后续接 Buttondown、Beehiiv、ConvertKit、Resend + Supabase。
  • 文案：Get notified when GTA 6 pre-orders go live。

7. UI 与交互要求

7.1 视觉风格

  • 英文界面。
  • 现代游戏工具站风格，不模仿 Rockstar 官方视觉。
  • 深色背景 + 高对比状态卡片。
  • 强调可读性，不要堆砌花哨动画。
  • 移动端优先，桌面端增强。

7.2 核心组件

组件 说明
CountdownTimer 实时倒计时组件
StatusCard 展示 Release Date、Price、Preorder、Platform 状态
PriceTable 展示不同地区、平台、版本价格
PreorderTable 展示各商店预购状态
HistoryTimeline 展示 GTA 系列发售时间线
ChartBlock 图表容器，用于时间间隔、评分、平台数据
PollWidget 价格预测/平台投票
FAQSection FAQ + Schema 结构化数据
Disclaimer 非官方声明

8.7 结构化数据

建议实现：

  • FAQPage Schema。
  • BreadcrumbList Schema。
  • VideoGame Schema 可选，但需要谨慎填写，只填确定信息。
  • WebSite Schema。
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "When is GTA 6 coming out?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "GTA 6 is officially set to release on November 19, 2026."
      }
    },
    {
      "@type": "Question",
      "name": "Is the GTA 6 price officially announced?",
      "acceptedAnswer": {
        "@type": "Answer",

      }
    }

}

8.8 SEO 注意事项

  • 不要制造虚假价格。
  • 每个页面都要有 Last updated。
  • 价格/预购页面需要明确区分 official、placeholder、rumor。
  • 图表必须配对应文字表格，避免搜索引擎读不到核心内容。
  • 站点上线后立即提交 Google Search Console 和 Bing Webmaster Tools。
  • 生成 sitemap.xml 和 robots.txt。

9. 变现设计

9.1 第一阶段：先不强插广告

MVP 阶段优先让页面被收录，提升加载速度和用户体验。广告位先预留，不要一开始堆广告。

9.2 Affiliate

  • 游戏主机：PS5、Xbox Series X|S。
  • 配件：手柄、耳机、游戏显示器、SSD、充电座。
  • 游戏周边：官方授权商品。
  • 实体游戏盘：等可靠预购链接出现后再放。

9.3 展示广告

  • 流量起来后可接 Google AdSense。
  • 广告位置建议：文章内容中段、FAQ 前、页面底部。
  • 不要遮挡核心倒计时和价格表。

9.4 邮件列表

邮件提醒是本站最有价值的沉淀方式。推荐表单文案： 

Get notified when GTA 6 pre-orders go live.
No spam. Only major GTA 6 price and pre-order updates.

10. 技术实现建议

10.1 推荐技术栈

方案 推荐程度 说明
Astro + Tailwind + JSON 最高 适合静态 SEO 站，速度快，部署简单
Next.js + Tailwind 高 适合后续扩展后端、API、邮件、数据库
Vite + React 中 做单页应用快，但 SEO 不如 Astro/Next 静态页

建议：第一版用 Astro。原因：SEO 友好、构建快、静态部署简单、适合 Cloudflare Pages。

10.2 部署

  • 代码托管：GitHub。
  • 部署平台：Cloudflare Pages。
  • 域名：避免强商标风险，不建议使用 official、rockstar、take-two 等词。
  • 分析：Cloudflare Web Analytics + Google Search Console + Bing Webmaster Tools。

10.3 性能要求

  • 首页 Lighthouse Performance 90+。
  • 首屏加载小于 2 秒。
  • 图片尽量不用大图，MVP 可不用图片。
  • 核心信息使用 HTML 文本，不要全部放在 Canvas 或图片里。
